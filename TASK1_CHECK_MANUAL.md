# 任务一核对和检查阅读手册

## 1. 先说结论

你现在看任务一，不用一上来就看代码。
你就按下面这个顺序看：

| 顺序 | 先看什么 | 为什么 |
|---|---|---|
| 1 | `output/task1_run_summary.csv` | 先确认哪些被试跑成功了 |
| 2 | `data/interim/s01/channel_table.csv` | 先确认通道是不是识别对了 |
| 3 | `data/interim/s01/trial_events.csv` | 看 trial 是不是切对了 |
| 4 | `output/figures/s01/psd_comparison.png` | 看滤波有没有起作用 |
| 5 | `output/figures/s01/waveform_comparison.png` | 看清洗前后差别 |
| 6 | `output/figures/s01/ica_eog_scores.png` | 看 ICA 是不是真的在找眼电 |
| 7 | `data/processed/s01/trial_qc.csv` | 看哪些 trial 肌电污染重 |
| 8 | `data/processed/s01/epochs_clean.fif` | 最后确认导出的成品对不对 |

## 2. 每一步展示什么

| 步骤 | 你应该展示什么 | 你应该怎么讲 |
|---|---|---|
| 数据导入 | `channel_table.csv` 截图 + 终端运行成功 | “我成功读到了 DEAP 原始 BDF，通道也分好了。” |
| 滤波 | `psd_comparison.png` | “滤波后低频漂移和杂乱频率更少了。” |
| 伪迹去除 | `waveform_comparison.png` + `ica_eog_scores.png` | “清洗后眨眼或大幅抖动的形状变小了。” |
| 切段 | `trial_events.csv` | “每个 trial 都有开始点、结束点、基线起点。” |
| 基线校正 | `epochs_clean.fif` 的形状和时间长度 | “我保留了 60 秒刺激期，前 3 秒只用来做校正。” |
| 质量检查 | `trial_qc.csv` | “哪些 trial 被肌电污染影响更重，后面建模时要留心。” |

## 3. 各个指标用大白话怎么理解

| 指标 | 大白话意思 | 你看它时关心什么 |
|---|---|---|
| 采样率 `128 Hz` | 每秒记录 128 个点 | 数据是不是被统一到后续好处理的速度 |
| 40 个 trial | 一个被试应该有 40 段视频刺激 | 数量对不对，不对就说明切段有问题 |
| 基线 3 秒 | 视频播放前的那一小段安静数据 | 用它把各段数据拉到差不多的起跑线 |
| PSD 频谱图 | 看不同频率的能量分布 | 滤波后杂乱的频率有没有变少 |
| ICA 成分 | 把混在一起的信号拆开看 | 被排掉的成分是不是像眼电 |
| muscle overlap seconds | 一个 trial 被肌电污染覆盖了多少秒 | 这个值越大，说明这段数据越脏 |
| epoch 形状 `40 x 32 x 7680` | 40 段数据，32 个 EEG 通道，每段 7680 个点 | 导出结果的结构是不是统一的 |

## 4. 你答辩时可以怎么说

| 步骤 | 一句话说法 |
|---|---|
| 导入 | “我先读取 DEAP 原始 BDF，然后把通道类型整理好。” |
| 滤波 | “我用 4-45 Hz 带通滤波，把太慢的漂移和太乱的高频杂质压下去。” |
| 去伪迹 | “我先用 ICA 找眼电，再用肌电标注看哪些片段更脏。” |
| 切段 | “我按视频开始和结束事件切 trial，不是随便硬切。” |
| 基线校正 | “我用视频前 3 秒把每段数据拉到同一个起点，这样更好比较。” |
| 导出 | “最后我保留了统一的 60 秒刺激期数据，方便后续特征提取和建模。” |

## 5. 最后核对表

| 你要检查的点 | 合格标准 |
|---|---|
| 能不能读到 BDF | 能 |
| 通道是不是分对 | EEG / EOG / EMG / stim 分得对 |
| trial 数是不是 40 | 是 |
| 每段刺激期是不是 60 秒 | 是 |
| 采样率是不是 128 Hz | 是 |
| 结果图是不是生成了 | 是 |
| `epochs_clean.fif` 是不是生成了 | 是 |
